# 临界态是否在追逃中有优势：场景与算法建议（讨论稿）

本文将研究思路拆成“可落地的假设—可测的指标—可复现实验—可迭代算法/场景”四条线，并给出更可能体现临界态优势的场景机制。

## 0. 当前假设设定（可在实验中调整）

- **场景**：2D 连续空间
- **信息约束**：逃跑者**不能显式通讯**（不能广播/发消息），只能通过局部感知看到邻居动作/位置，以及看到追捕者/环境线索
- **追捕者能力**：允许做预测拦截与协作，但实验路线前期从简单策略开始逐步增强
- **目标**：集群优势可以体现在多个指标上（存活数、成功率、到达时间等），因此建议同时报告一组互补指标
- **临界性证据**：以能说服审稿人的证据链为准（下文给一个相对稳健的最低配置）

## 1. 研究假设（建议写成可检验的形式）

**H1（参数扫描）**：当逃跑者的“社会耦合强度/从众权重” `w` 接近集群的有序-无序相变点时，追逃任务指标（例如成功率、存活数、到达时间）达到最优或接近最优。

**H2（机制解释）**：近临界态在以下权衡上更优：
- **快速一致**（信息传播/集体转向快） vs **保持多样性**（不过早锁死在错误方向、能分裂探索）。
- **抗扰稳定**（不被追捕者轻易撕裂） vs **可塑反应**（遇到威胁能快速重组与转向）。

**H3（SOC）**：通过局部负反馈调参，可让集群在外部驱动（追捕压力/环境扰动）和耗散（捕获/边界/碰撞）下自组织到（准）临界，从而在不同场景变化下保持鲁棒性能。

## 2. 基线“集群逃跑算法”候选（用于参数扫描）

“从完全跟随邻居到完全不跟随邻居”的旋钮，最自然的实现方式是把它做成对**邻居对齐/凝聚项**与**个体目标/规避项**的加权。

### 2.1 Vicsek/对齐模型变体（最直接）

状态：位置 `x_i`、速度方向 `θ_i`（或速度向量 `v_i`），每步更新方向为：

- 邻居对齐：朝邻域内平均朝向
- 威胁规避：远离追捕者（或其预测位置）
- 目标驱动：朝安全区线索/探索方向
- 噪声：角噪声（强度可固定或做 SOC）

旋钮建议：
- `w_align ∈ [0,1]`：对齐权重（控制“跟随邻居程度”）
- 或 `noise`：噪声强度也能产生有序-无序转变（很多模型里噪声是相变控制参）

优点：临界点和统计指标相对成熟，容易先做出“临界区 vs 性能峰”的对照。

### 2.2 Couzin zonal model（分区规则）

把邻居影响拆成三个区：排斥区（防碰撞）、对齐区、吸引区。可调参：
- 吸引/对齐相对强度，或各区半径比
- 或“跟随邻居”的概率（随机从众 vs 自主）

优点：更符合动物群体行为直觉，分裂-重组更自然。

## 3. 如何“识别临界态”（不依赖单一指标）

建议用一组互补指标，而不是只看一个“看起来像幂律”的图。

### 3.1 有序参数 + 易感性（最稳的起点）

- 有序参数：极化度 `P(t)=|Σ v_i|/N`（或速度一致性）
- 易感性：`χ = N * Var(P)` 或 `Var(P)` 随控制参变化是否出现峰值

在参数扫描 `w` 或 `noise` 时，若 `P` 出现从高到低的转变、同时 `χ` 在某区域峰值明显，这通常对应临界附近。

### 3.2 相关长度/相关时间（更“临界”）

在速度/朝向涨落上计算空间相关函数 `C(r)`，拟合相关长度 `ξ`，观察 `ξ` 是否在某参数区显著增大（有限尺寸下会饱和）。

### 3.3 “雪崩/级联”统计（SOC 路线更关键）

定义一个“扰动事件”（例如某个体因近距离追捕触发急转/加速），用阈值把后续一段时间内被牵连的个体数/总转向量定义为事件规模 `S`，统计 `P(S)` 是否近幂律、是否跨尺度。

提示：追逃任务本身就是持续驱动，捕获/边界是耗散，很适合做 SOC 的“驱动-耗散”框架。

## 4. 更可能体现临界优势的追逃场景设计（场景选题）

“安全区未知 + 容量限制”能逼迫集群做“探索-利用”和“分裂-一致”的动态权衡，适合作为检验临界优势的任务骨架。

### 首选建议（最适配“无显式通讯”的信息传播机制）

在“只能看邻居动作、不能显式通讯”的约束下，推荐把“信息传播”设计成**只能靠模仿/从众扩散**：

- 地图中有多个安全区 `G_k`（位置固定），每个有容量 `cap_k`（可不同）。
- 安全区对逃跑者是**局部可见**：只有进入感知半径 `r_detect`（或进入某片区域）才知道“这是安全区 + 剩余容量/是否可用”。
- 逃跑者个体一旦发现安全区，会自然地朝它移动；其他个体只能通过“看到邻居朝某方向出现一致运动”来间接获得信息。

这样一来，`w_align`（从众）直接控制“发现信息后能否快速扩散成群体共识”，同时也影响“是否保留少量偏离者继续探索/避免容量耗尽后集体拥堵”。这正是临界态可能占优的点：**在快速共识与多样性之间取得最好平衡**。

下面按“机制”给出 4 套场景，可从简单到复杂逐步叠加。

### 场景 A：多安全区 + 容量限制（最推荐的基准）

- 2D 连续空间或栅格都可以，多个安全区 `G_k`，每个有容量 `cap_k`（可能不同）。
- 逃跑者初始不知道哪个安全区可用（或不知道容量），只有进入一定感知半径后才知道“这里是安全区以及剩余容量”。
- 成功指标：时间窗 `T` 内进入安全区的人数、未被捕人数、平均到达时间；并衡量“拥挤浪费”（很多人挤满后才改道）。

为什么可能需要临界态：
- 需要一部分人继续探索，另一部分迅速形成共识冲向已确认的安全区；
- 需要快速分裂（多安全区并行）与快速重组（发现容量不足时撤离）。

### 场景 B：移动/不确定的安全区（对信息传播更敏感）

- 安全区以未知策略移动（或只在某时间段出现）。
- 个体只能在局部感知到安全区“信标强度”，且噪声较大。

临界优势点：
- 高相关长度让弱信号更容易在群体中放大并形成一致移动，但又不能过度刚性导致跟错。

### 场景 C：诱饵/假安全区（测试“不过早锁死”）

- 设定部分区域发出假信号，靠近后才发现不可进入/会被捕获更容易。

临界优势点：
- 近临界能在形成主流方向的同时保留少量偏离者，降低被诱饵“一锅端”的风险。

### 场景 D：狭窄通道 + 追捕者阻断（测试分裂重组与韧性）

- 地图有瓶颈、通道、障碍；追捕者可以“卡位”。
- 逃跑者必须在拥挤-分流之间权衡。

临界优势点：
- 太有序会导致拥挤踩踏式阻塞；太无序又会被逐个击破。临界附近可能最平衡。

## 5. 自组织临界（SOC）策略：怎么“只靠局部观测”调参

核心是做一个**负反馈自调节**，让系统被驱动时保持在“既不冻结也不崩散”的边界附近。实际落地时，可先将 SOC 实现为“自组织准临界”（SOqC）。

### 5.0 为什么 SOC 在“不能显式通讯”条件下更有意义

在不能显式通讯的约束下，个体难以依赖“全局最优信息”做决策；SOC/自调参将“适应环境变化”的责任下放到局部反馈：当追捕强度、初始分布或安全区容量分布变化时，个体可依靠局部统计量把群体推回到“既能传播信息又不至于锁死”的状态附近。

### 5.1 可调的旋钮（建议选 1 个先做）

1) **对齐权重** `w_align`：越大越像“从众”，越小越自主  
2) **角噪声强度** `η`：越大越无序  
3) **邻域半径/邻居数**：改变相互作用拓扑  

### 5.2 个体可用的局部观测（不需要全局量）

- 本地极化：邻域内朝向一致性 `P_local`
- 本地涨落：邻域朝向方差 `Var_local(θ)` 或速度方差
- 近邻数/密度：`k_i`、平均距离 `d_nn`
- 威胁强度：最近追捕者距离/相对速度

### 5.3 一个简单但有效的自调参规则（负反馈）

以调 `w_align` 为例（示意）：

- 若 `P_local` 太高且 `Var_local` 太低：说明局部“过于刚性/冻结”，降低 `w_align` 或增加噪声；
- 若 `P_local` 太低且集群破碎风险上升（`k_i` 很低或 `d_nn` 很大）：提高 `w_align` 或降低噪声；
- 威胁强时（追捕者近）：短时提高对齐/凝聚以避免被撕裂；威胁弱时降低对齐以促进探索。

这类规则的目标不是“把 `P_local` 调到某个常数”，而是把系统维持在“涨落较大但不崩盘”的边界附近；可将目标写成：让 `Var_local` 在一个中等偏高区间，并约束 `k_i` 不低于某阈值。

### 5.4 更像 SOC 的“级联目标”（进阶）

如需把“雪崩事件”定义清楚（例如由追捕触发的急转传播），可让个体调整参数使得：
- 小事件常见、大事件少但存在（跨尺度）
- 事件规模分布在对数坐标下近线性（近幂律）

实践上可以先用更简单的代理目标：让“被牵连个体数”的均值接近某个分界（类似分支比 ~1 的思想）。

## 6. 建议的最小闭环实验路线（不容易走偏）

1) **先做逃跑者无追捕的相变定位**：在空场景中扫描 `w_align`（或 `η`），画 `P`、`Var(P)`、相关长度，确定近临界区域。
2) **再上追逃场景 A（多安全区+容量）**：保持追捕者策略简单（例如直线追最近目标或预测拦截），比较不同 `w` 的成功率/存活数。
3) **检验“性能峰 vs 临界峰”是否重合**：不重合也很有价值（说明临界不是必要条件，或指标/场景不匹配）。
4) **最后做 SOC 版自调参**：在场景变化（追捕者数量、地图、信号噪声）下看是否比固定最优参数更鲁棒。

### 6.1 追捕者策略的递进（从易到难，避免一上来太强）

- **P0 最近邻追逐（baseline）**：每个追捕者追最近的逃跑者（带速度上限/转向上限）。
- **P1 预测拦截（单体更强）**：基于相对速度估计拦截点（可先用一阶线性预测/比例导航）。
- **P2 协作分配（多体更强）**：追捕者之间用代价矩阵（距离或预计拦截时间）做目标分配（匈牙利算法或贪心近似），减少重复追同一目标。
- **P3 阻断/围堵（场景化更强）**：在瓶颈/安全区附近做占位与分割（建议在场景 A 跑通后再引入）。

该递进用于观察：近临界优势是否只在追捕弱时出现，还是在追捕增强后仍保持（这对论文说服力很关键）。

## 7. 审稿可接受的证据链（建议最低配置）

为避免“看起来像幂律/像临界但其实是有限尺寸或阈值效应”的质疑，建议用这套相对稳健的组合：

1) **相变定位（无追捕或弱追捕）**：`P(w)` 转变 + `χ(w)=N·Var(P)`（或 `Var(P)`）出现峰值；至少做 2~3 个不同 `N`（有限尺寸）报告峰值位置与不确定性。
2) **相关性证据（加分且更“临界”）**：速度/朝向涨落的空间相关函数 `C(r)` 给出相关长度 `ξ(w)`，在同一区域显著增大（有限尺寸饱和也可接受，但需报告）。
3) **任务性能对齐（核心结论）**：同一组 `w` 上报告追逃指标（存活率/成功率/到达时间分位数/拥挤浪费等），明确性能峰与 `χ/ξ` 峰的关系（重合/偏移/不相关都可，但要有机制解释）。
4) **稳健性（避免“挑参数”质疑）**：多随机种子置信区间；追捕者从 P0→P2 的递进下结论是否保持；至少一种场景变化（如容量分布或安全区数量变化）。

若主打 SOC：补充“自调参”在场景变化下能维持较高 `χ/ξ` 且性能退化更小，并与“固定最优 `w*`”对比鲁棒性。

## 8. 实现优先级建议（可选默认）

- **主场景**：采用“首选建议”，即**多安全区 + 容量限制 + 局部可见**作为基准；再加入“诱饵/假安全区”作为加强实验（更容易体现“不过早锁死”的价值）。
- **追捕强度标定**：速度比 `v_p/v_e` 建议从 `1.05~1.20`（略强但不碾压）起步，并在后续稳健性实验中做网格化扫描。
- **边界与障碍**：建议先用“无障碍 + 周期边界”跑通相变定位，再加入“有限边界/障碍”做任务化展示与稳健性检验。
