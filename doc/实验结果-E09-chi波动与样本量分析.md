# E09 图4 波动分析：`chi` 高区间下 `safe` 离散度来源与样本量判断

## 1. 问题

观察到 E09（`results_20260206_walign_pressure_091314_sr14_240seeds`）的 `safe`–`chi` 散点在 `chi≈4.5` 附近出现较大纵向离散，需判断：

1. 该离散是否主要由样本量不足导致；
2. 该离散是否由实验设计中的混合条件（不同追捕压力）导致；
3. 在固定追捕压力下，`w_align` 的影响是否本身就较弱。

## 2. 数据与方法

- 数据：
  - 聚合表：`doc/results_20260206_walign_pressure_091314_sr14_240seeds/group_summary.csv`
  - 原始运行：`runs/sweep_20260206_merged_pressure_sr14_240_grid/results.csv`
- E09 设计：
  - `speed_ratio ∈ {0.9, 1.3, 1.4}`
  - `w_align ∈ {0.00, 0.05, ..., 1.00}`
  - seeds：`sr=0.9/1.3` 为 120，`sr=1.4` 为 240
  - 每个格点统计 `safe_mean` 与 `chi_mean`。
- 诊断方法：
  - 按 `speed_ratio` 分层后再看 `safe`–`chi`；
  - 对 `chi ∈ [4.2, 4.8]` 做方差分解（between-speed-ratio vs within-speed-ratio）；
  - 用原始 `results.csv` 做分层 bootstrap（每个 `w_align` 内重采样）估计 `corr(safe,chi)` 的不确定性。

## 3. 核心结果

### 3.1 `chi≈4.5` 的大波动主要来自“混合了不同压力条件”

- 在 `chi ∈ [4.2, 4.8]` 的 34 个格点中，整体 `safe` 范围为 `[0.2510, 0.3946]`（看起来很“散”）。
- 但按 `speed_ratio` 分层后：
  - `sr=0.9`：`safe ∈ [0.3598, 0.3946]`
  - `sr=1.3`：`safe ∈ [0.2783, 0.3064]`
  - `sr=1.4`：`safe ∈ [0.2510, 0.2791]`
- 结论：同样 `chi` 水平下，不同追捕压力给出不同生存率基线，叠在一张图上会形成显著“竖向厚度”。

方差分解（`chi ∈ [4.2, 4.8]`）：

- `safe` 总方差：`0.002411`
- 由 `speed_ratio` 组间差异解释：`0.002331`
- 组内方差：`0.000080`
- 组间占比：`96.7%`

即该区间离散的主导因素是“压力层混合”，不是同一层内随机抖动。

### 3.2 “样本量不足”不是主要原因

分层 bootstrap（基于原始运行重采样）得到 `corr(safe,chi)` 的 95% 区间：

- `sr=0.9`：`0.272`，95% CI `[0.037, 0.409]`
- `sr=1.3`：`0.378`，95% CI `[0.098, 0.477]`
- `sr=1.4`：`0.024`，95% CI `[-0.148, 0.200]`

解读：

- `sr=0.9/1.3` 下，正相关方向在重采样下基本稳定；
- `sr=1.4` 下相关接近 0，且区间跨 0，说明“在高压条件下 `chi` 对 `safe` 的解释力弱”；
- 这更像机制问题（强压下指标解耦），而不是 seeds 不足导致符号不稳。

### 3.3 在固定压力下，`w_align` 的影响是“中等偏弱”，且随压力增强而减弱

固定 `speed_ratio` 看 `safe_mean` 随 `w_align` 的变化幅度：

- `sr=0.9`：范围约 `0.049`
- `sr=1.3`：范围约 `0.028`
- `sr=1.4`：范围约 `0.028`

固定压力内线性解释度（`safe ~ chi`）：

- `sr=0.9`：`R^2 ≈ 0.074`
- `sr=1.3`：`R^2 ≈ 0.143`
- `sr=1.4`：`R^2 ≈ 0.001`

说明：

- `w_align` 并非“完全没影响”，但在高压力（`sr=1.4`）下，`chi` 已不能单独解释 `safe` 的差异；
- 该条件下更可能由其他结构变量共同决定（如 `xi_fluct`、连通分量数、局部碎裂形态）。

### 3.4 原始点云层面：`chi` 与生存率关系确实很弱

不按 `w_align` 或 seed 聚合，直接使用单次运行数据（`results.csv`）计算：

- 全部原始点：`corr(chi, safe_frac) ≈ -0.016`
- `sr=0.9`：`≈ -0.025`
- `sr=1.3`：`≈ 0.051`
- `sr=1.4`：`≈ 0.018`

结论：原始点云层面基本看不出稳定单调关系。  
这与“格点均值层面（多 seed 聚合）可见一定结构”并不矛盾，二者反映的是不同层级的信息：

1. 原始层：轨迹随机性和离散事件主导；
2. 聚合层：参数效应在随机背景上的平均趋势。

## 4. 对图4 E09的直接结论

1. `chi≈4.5` 处看起来“波动很大”，主要是把 `sr=0.9/1.3/1.4` 混在一起画造成的；  
2. 在固定 `speed_ratio` 后，波动明显收敛；  
3. 当前 seeds 规模（120/240）足以支持“高压下 `chi`–`safe` 解耦”的判断；  
4. 对齐强度在高压场景中影响变弱，出现“改 `w_align` 但 `safe` 提升有限”的平台效应。

## 4.1 为什么 `chi<4` 点少，而 `chi>4` 点多且“看起来更乱”

### 点数不均衡是指标分布与参数覆盖共同造成的

- E09 共 63 个格点，其中 `chi<4` 仅 14 个，`chi>=4` 有 49 个。
- 按压力分层：
  - `sr=0.9`：`chi>=4` 为 `16/21`
  - `sr=1.3`：`chi>=4` 为 `14/21`
  - `sr=1.4`：`chi>=4` 为 `19/21`
- 这意味着绝大多数 `w_align` 格点都落在 `chi` 的高区，低区只剩少量点（主要是很低 `w_align`）。

### 机制解释：`chi` 在高区形成“平台”，同一 `chi` 对应多个不同条件

- 本实验中 \(\chi=N_e\cdot\mathrm{Var}(P)\)，且 `N_e=128`，`chi=4` 对应 `P_var=0.03125`。
- E09 中大量格点的 `P_var` 都在 `0.031~0.039` 附近，导致 `chi` 聚集在 `4~5` 区间（横轴拥挤）。
- 当横轴拥挤且叠加不同 `sr` 的基线差异时，同一 `chi` 会对应明显不同的 `safe`，视觉上表现为“高 `chi` 区很乱”。

### 因此，“乱”不等于“没有结构”

- 结构存在于分层后：在相同 `sr` 下，`safe` 范围明显收敛；
- 结构也存在于机制上：`sr` 越高，`safe` 基线越低，且 `chi` 对 `safe` 的解释力越弱（尤其 `sr=1.4`）。
